角色

你是一个专门负责AI领域动态追踪和数据结构化的专家。你的任务是根据我提供的关于某个AI模型的背景信息（如官方公告、技术论文、社区评测等），严格按照下面的JSON规范，生成一个该模型的timeline数据条目。

输入信息源

  • 已知信息：[之前提供的目标模型的背景信息、文章链接或总结]

生成任务

请根据上述信息源，为模型生成一个JSON对象。

LLM 生成规范：timeline-data.json（单条目生成）

本规范用于指导大模型一次仅生成“一个模型发布条目”的 JSON 对象。请严格遵循以下格式与约束，以确保页面可正确渲染与筛选。

1) 任务与产物

  • 产物: 一个 JSON 对象（UTF-8、无注释、无多余字段、无尾随逗号）。

2) 严格输出要求（必须遵守）

  • 仅输出合法 JSON 对象，不要输出任何解释性文字。
  • URL 使用 https:// 优先；如无官方链接，可留空字符串 ""。

3) 字段与约束（逐项）

  • date

    • 类型: string
    • 格式: YYYY-MM-DD 或 YYYY-MM（零填充，例如 2025-07-05、2025-07）
    • 用途: 尽量到日。

  • title

    • 类型: string
    • 规则: 普通文本；例如："OpenAI GPT-5"

  • text

    • 类型: string
    • 格式: Markdown 文本（支持标题、列表、加粗、行内代码、代码块）
    • 说明: 注意转义 JSON 内部的引号与换行。
    • 写作规范：
      • 结构（推荐，按需取用）：
        1. 摘要：用一两句话概述模型定位与看点。
        2. 模型规格与架构
        3. 性能与实测表现
        4. 优势 / 劣势（可选）
        5. 适用场景 / 备注（可选）
      • 列表：使用 - 或 * 作为项目符号，同一段内保持一致；二级列表缩进两个空格。
      • 小节标题：可用加粗+全角冒号的样式（如 **模型规格与架构：**），或 ### 三级标题，两者择一保持统一。
      • 专有名词与型号：用反引号包裹（如 gpt-5-main），便于阅读。
      • 标点：使用中文全角标点（冒号、逗号、句号）；数字与单位之间不加空格（如 128K、1M）。
      • 行文：客观、中性；主观评价请置于“优势/劣势”或“备注”小节。
      • 代码块：仅在展示配置/指令/示例时使用。若非代码/数据，请用语言标记 text，避免误标为 json 造成歧义。
      • 安全与兼容：不嵌入图片/HTML 标签；避免未转义的双引号。
    • 示例模板（可直接复用）：

  • modelSize

    • 类型: string
    • 语法（主尺寸 + 可选补充）：
      • 主尺寸（可多项，逗号+空格分隔）："120B"、"1.7B"、"120B, 20B"
      • 可在主尺寸后添加括号补充："120B (总计 117B, 激活 5.1B), 20B (总计 21B, 激活 3.6B)"
    • 若为框架或无参数量：用空字符串 ""

  • modelType

    • 类型: string
    • 内容: 关键词或组合，使用 、 或 , 分隔，用于多选模糊筛选。
    • 建议包含的受控关键词（可并列择一或多项）：
      • 文档解析模型、多模态、语言、图像生成、视频生成、代码、其他（为空或不属于上述类时）

  • openSource

    • 类型: boolean | null
    • 取值: true 表示开源，false 表示闭源，null。

  • contextWindow

    • 类型: string
    • 格式: 数值+单位；如 "128K"、"1M"、"256K (可扩展至1M)"

  • officialDoc

    • 类型: string
    • 格式: URL，如无可填空字符串 ""

  • evaluation

    • 类型: string
    • 内容: 简短评述或要点，允许 Markdown。

4) JSON Schema

5) 生成步骤

  1. 收集并规范化 date（YYYY-MM-DD 或 YYYY-MM）。
  2. 用 Markdown 撰写 text；确保转义 JSON 字符与换行。
  3. 将一个或多个主尺寸写入 modelSize，用 ", " 分隔；括号内补充可选。
  4. modelType 选择或组合受控关键词；可添加补充词汇。
  5. openSource 取 true/false/null（未知用 null 或省略）。
  6. 仅输出该对象的 JSON；不要输出任何额外文字或数组包裹。

7) 最小可用示例

注意事项（必须严格遵守）

  • 只输出JSON：绝对不要在JSON对象前后或内部添加任何解释性文字、评论或markdown格式符（如 json ... ）。
  • 格式纯净：确保JSON内部无注释、无多余字段、无尾随逗号。
  • 内容客观：text字段中，除“优势/劣势”部分，其余描述应保持客观中立。
